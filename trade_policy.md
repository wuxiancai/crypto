# 交易策略政策

更新时间：2026-07-01

## 目标口径

本项目第一版只运行 Paper Trading、Backtest、Web 状态页和复盘，不开发真实下单或测试网下单。

当前策略只允许三种时间线仓位：

- `WEEKLY`：周线单。
- `DAILY`：日线单。
- `H4`：4H 单。

三种时间线各自拥有独立策略，独立决定开仓、止盈、止损、减仓和退出。不同时间线之间不互相过滤、不互相阻断、不互相强制平仓。

同一交易对可以同时持有不同时间线的反向仓。例如 BTCUSDT 可以同时持有周线多、日线空和 4H 多。

同一交易对的同一时间线只能持有一个方向：

- 同一时间线已有多单时，不允许再开空单。
- 同一时间线已有空单时，不允许再开多单。
- 同一时间线已有同方向仓位时，可以追加仓位。

所有时间线仍共享账户级风控：

- Kill Switch。
- 最大回撤熔断。
- 总计划风险上限。
- 总名义价值上限。
- Liquidation Guard。
- Stop Order Guard。
- 手续费 / 计划风险过滤。

## 结构方向与风险预算

“参考上级方向”只用于定义当前单的结构属性和风险预算，不用于阻断交易。

结构风险和风险预算分开表述：

- 顺上级方向：结构风险较低，允许较高风险预算。
- 逆上级方向：结构风险较高，只允许较低风险预算。

风险预算表示单笔允许亏损额度或仓位预算，不表示交易本身更危险。

## 周线单策略

周线单只使用周线数据生成信号，不依赖日线或 4H 确认。

### 周线多单触发条件

满足以下条件时，可以生成 `WEEKLY_LONG_TREND`：

- 周线 EMA15 > 周线 MA60。
- 周线 EMA15 斜率 > 0。
- 周线 ADX >= 最小趋势强度阈值。
- 周线 DI+ > DI-。
- 当前没有 `WEEKLY` 空单。

如果已有 `WEEKLY` 多单，且再次出现同方向有效信号，可以追加周线多仓，但追加仍受账户级风控约束。

### 周线空单触发条件

满足以下条件时，可以生成 `WEEKLY_SHORT_TREND`：

- 周线 EMA15 < 周线 MA60。
- 周线 EMA15 斜率 < 0。
- 周线 ADX >= 最小趋势强度阈值。
- 周线 DI- > DI+。
- 当前没有 `WEEKLY` 多单。

如果已有 `WEEKLY` 空单，且再次出现同方向有效信号，可以追加周线空仓，但追加仍受账户级风控约束。

### 周线止损与减仓

周线单不使用日线或 4H 信号止损。

周线初始防线：

- 周线多单：默认使用周线 MA60 或周线结构低点中更符合风控的一侧作为防线。
- 周线空单：默认使用周线 MA60 或周线结构高点中更符合风控的一侧作为防线。

周线分批减仓条件：

- 趋势防线破坏：收盘价有效跌破 / 突破周线 MA60。
- 结构破坏：收盘价有效跌破周线结构低点，或突破周线结构高点。
- 动能破坏：ADX 低于阈值，或 DI 方向反转。

每类破坏在同一笔周线单生命周期内最多触发一次减仓。

### 周线退出

周线完全退出条件：

- 周线多单出现完整空头反转：EMA15 < MA60，EMA15 斜率 < 0，ADX 达标，DI- > DI+。
- 周线空单出现完整多头反转：EMA15 > MA60，EMA15 斜率 > 0，ADX 达标，DI+ > DI-。
- 强平保护触发。
- Kill Switch 要求关闭仓位。

周线单不使用普通 2R 固定止盈或 4H ATR 移动止盈。

## 日线单策略

日线单只由日线策略决定开仓、止盈、止损、减仓和退出。周线方向只用于定义日线单是主方向单还是反弹单，并影响风险预算。

### 日线方向分类

当周线方向为空：

- 日线做空 = `DAILY_SHORT_TREND`，日线主方向空单，结构风险较低，风险预算较高。
- 日线做多 = `DAILY_LONG_REBOUND`，日线反弹多单，结构风险较高，风险预算较低。

当周线方向为多：

- 日线做多 = `DAILY_LONG_TREND`，日线主方向多单，结构风险较低，风险预算较高。
- 日线做空 = `DAILY_SHORT_REBOUND`，日线反弹空单，结构风险较高，风险预算较低。

当周线方向不明确：

- 日线仍可独立交易。
- 日线交易标记为 `DAILY_*_NEUTRAL` 或在原因中记录 `weekly neutral`。
- 风险预算使用中性预算，不高于主方向预算。

### 日线多单触发条件

满足以下条件时，可以生成日线多单：

- 日线 EMA15 > 日线 MA60。
- 日线 EMA15 斜率 > 0。
- 日线 ADX >= 最小趋势强度阈值。
- 日线 DI+ > DI-。
- 当前没有 `DAILY` 空单。

如果已有 `DAILY` 多单，且再次出现同方向有效信号，可以追加日线多仓。

### 日线空单触发条件

满足以下条件时，可以生成日线空单：

- 日线 EMA15 < 日线 MA60。
- 日线 EMA15 斜率 < 0。
- 日线 ADX >= 最小趋势强度阈值。
- 日线 DI- > DI+。
- 当前没有 `DAILY` 多单。

如果已有 `DAILY` 空单，且再次出现同方向有效信号，可以追加日线空仓。

### 日线止损

日线单只使用日线结构和日线指标止损：

- 日线多单：初始止损优先使用日线结构低点；结构低点不可用时使用日线 ATR 防线。
- 日线空单：初始止损优先使用日线结构高点；结构高点不可用时使用日线 ATR 防线。

日线止损不由周线或 4H 触发。

### 日线止盈与退出

日线止盈：

- 触达 2R 后不立即全部平仓。
- 至少保护 1R 利润。
- 剩余仓位按日线 ATR 动态移动止盈。

日线完全退出条件：

- 日线多单出现完整日线空头反转。
- 日线空单出现完整日线多头反转。
- 日线止损触发。
- 日线 ATR 移动保护线触发。
- 强平保护触发。
- Kill Switch 要求关闭仓位。

## 4H 单策略

4H 单只由 4H 策略决定开仓、止盈、止损、减仓和退出。日线方向只用于定义 4H 单是主方向单、反弹单或中性单，并影响风险预算。

### 4H 方向分类

当日线方向为空：

- 4H 做空 = `H4_SHORT_TREND` / `H4_SHORT_BREAKOUT` / `H4_SHORT_CONTINUATION`，4H 主方向空单，结构风险较低，风险预算较高。
- 4H 做多 = `H4_LONG_REBOUND` / `H4_LONG_BREAKOUT`，4H 反弹多单，结构风险较高，风险预算较低。

当日线方向为多：

- 4H 做多 = `H4_LONG_TREND` / `H4_LONG_BREAKOUT` / `H4_LONG_CONTINUATION`，4H 主方向多单，结构风险较低，风险预算较高。
- 4H 做空 = `H4_SHORT_REBOUND` / `H4_SHORT_BREAKOUT`，4H 反弹空单，结构风险较高，风险预算较低。

当日线方向不明确：

- 4H 仍可独立交易。
- 4H 交易标记为中性。
- 风险预算使用中性预算，不高于主方向预算。

### 4H 多单触发条件

满足以下条件时，可以生成 4H 多单：

- 4H EMA15 > 4H MA60。
- 4H EMA15 斜率 > 0。
- 4H ADX >= 最小趋势强度阈值。
- 4H DI+ > DI-。
- BOLL 宽度达到最小开口阈值。
- 出现 4H 结构突破、回踩确认或顺势延续形态。
- 当前没有 `H4` 空单。

如果已有 `H4` 多单，且再次出现同方向有效信号，可以追加 4H 多仓。

### 4H 空单触发条件

满足以下条件时，可以生成 4H 空单：

- 4H EMA15 < 4H MA60。
- 4H EMA15 斜率 < 0。
- 4H ADX >= 最小趋势强度阈值。
- 4H DI- > DI+。
- BOLL 宽度达到最小开口阈值。
- 出现 4H 结构跌破、回抽确认或顺势延续形态。
- 当前没有 `H4` 多单。

如果已有 `H4` 空单，且再次出现同方向有效信号，可以追加 4H 空仓。

### 4H 止损

4H 单只使用 4H 结构和 4H 指标止损：

- 4H 多单：初始止损优先使用 4H 结构低点；结构低点不可用时使用 4H ATR 防线。
- 4H 空单：初始止损优先使用 4H 结构高点；结构高点不可用时使用 4H ATR 防线。

4H 止损不由周线或日线触发。

### 4H 止盈与退出

4H 止盈：

- 触达 2R 后不立即全部平仓。
- 至少保护 1R 利润。
- 剩余仓位按 4H ATR 动态移动止盈。

4H 完全退出条件：

- 4H 多单出现完整 4H 空头反转。
- 4H 空单出现完整 4H 多头反转。
- 4H 止损触发。
- 4H ATR 移动保护线触发。
- 强平保护触发。
- Kill Switch 要求关闭仓位。

## Backtest 与 Paper 一致性

Backtest 和 Paper 必须共享同一套策略政策：

- 同样只运行 `WEEKLY / DAILY / H4` 三种时间线单。
- 同样允许不同时间线之间反向共存。
- 同样禁止同一时间线内多空共存。
- 同样允许同一时间线同方向追加仓位。
- 同样按上级方向计算结构属性和风险预算。
- 同样按各自时间线的 K 线事件执行开仓、止损、止盈、减仓和退出。

任何 Web 展示、复盘事件、回测归档和运行日志，都必须能看出：

- 当前仓位属于哪个时间线。
- 当前仓位是主方向、反弹还是中性。
- 当前仓位使用了哪个风险预算。
- 当前仓位的止损、止盈和退出依据来自哪个时间线。
