# Multi-Timeframe Trading System

## 1. 文档目的

本文档用于明确交易系统的大防线和仓位层级。它先定义“什么级别的仓位在什么环境下可以存在”，再逐步补充入场、止损、止盈、减仓和风险约束。

这不是最终参数文档。后续所有“死叉、反弹失败、BOLL 开口、结构突破、趋势破坏、动能破坏”等词，都必须继续量化，才能进入回测或自动执行。

核心目标：

- 周线决定大环境，也对应最高级别的周线仓。
- 日线在周线环境下决定战术仓。
- 4H 只负责执行、过滤和 breakout 捕捉。
- 风险预算统一管理，仓位不能简单叠加。

---

## 2. 仓位只保留三类

系统只保留三类仓位：

```text
WEEKLY = 周线仓
DAILY  = 日线仓
H4     = 4H 仓
```

不再拆出过多仓位名称。方向、顺逆势、反弹或突破，都作为仓位属性记录，而不是新增仓位类型。

示例：

```text
level = WEEKLY
side = SHORT
context = weekly_bear + daily_confirmed

level = DAILY
side = LONG
context = rebound_under_weekly_bear

level = H4
side = SHORT
context = breakout_under_weekly_bear
```

这样可以避免策略命名膨胀，也方便后续统一做仓位管理、复盘和风控。

---

## 3. 核心定义

### 3.1 周线不是单纯过滤器

周线既是趋势状态，也是仓位层。

当周线进入空头或多头状态，并获得日线确认时，系统可以建立周线仓。周线仓计划持有更长周期，但不代表必须死扛到金叉或死叉才处理。

### 3.2 日线是周线环境内的战术层

日线不能直接推翻周线趋势。日线只能在周线环境下产生战术仓。

日线仓有两种模式：

- 反弹模式：例如周线空趋势下的日线反弹多单。
- 顺势模式：例如周线空趋势下的顺势日线空单。

同一交易对、同一周线环境下，日线仓不能同时存在反弹单和顺势单。要么持有反弹单，要么持有顺势单。反弹单结束后，才允许根据规则切换为顺势单。

### 3.3 4H 是执行层

4H 负责提高入场质量，也负责捕捉趋势初期 breakout。

4H 不决定大趋势，不改变周线仓身份，也不应该制造频繁交易。若 4H 处于窄幅震荡、BOLL 未开口、结构不清晰，系统宁可不开 4H 仓。

### 3.4 仓位比例不是账户资金比例

本文中的 60%、30%、10% 是计划仓位预算的层级分配，不等于直接使用账户资金比例，也不等于保证金比例。

落地时必须转换为：

- 单笔计划亏损。
- 总计划亏损。
- 保证金占用。
- 名义价值。
- 杠杆倍数。
- 强平安全距离。

---

## 4. 总体分层

```text
Weekly = 大趋势状态 + 周线仓
Daily  = 周线环境下的日线仓
H4     = 执行过滤 + breakout 捕捉
Risk   = 统一预算与防线
```

周线是第一防线。它决定系统处于牛市、熊市还是转换期，并决定是否允许建立周线仓。

日线是第二防线。它确认周线仓是否可以开仓，也负责在周线趋势内识别反弹、反弹失败和顺势恢复。

4H 是第三防线。它只在波动结构展开、BOLL 开口、结构突破或 breakout 形成时参与。

---

## 5. 仓位身份表

| 仓位层级 | 允许模式 | 预算层级 | 持仓目标 | 互斥规则 |
| --- | --- | --- | --- | --- |
| `WEEKLY` | 周线趋势仓 | >= 60% | 吃周线趋势 | 同一交易对只保留一个周线方向 |
| `DAILY` | 反弹单 / 顺势单 | >= 30% | 做日线波段 | 反弹单和顺势单不能同时存在 |
| `H4` | breakout / pullback / continuation | <= 10% | 做执行级机会 | 只在严格条件下存在 |

仓位身份一旦建立，后续止盈、止损、减仓、平仓必须按该层级执行。不能因为短期波动，把周线仓当日线仓处理，也不能把日线反弹单升级为周线仓。

各层级必须使用独立的止损止盈生命周期：

- `WEEKLY` 仓只接受周线级别的趋势防线、结构防线、动能防线和周线反向金叉/死叉管理。
- `DAILY` 仓使用日线波段止损止盈，不能反向影响已经存在的周线仓。
- `H4` 仓使用 4H 执行级止损止盈，只管理自身 breakout / pullback / continuation 仓位。

各层级也必须使用独立的杠杆、保证金和强平安全模型，不能用一个全局杠杆处理所有仓位。默认落地口径：

- `WEEKLY`：低杠杆、宽防线，默认 2X，适配周线 MA60、周线结构和周线动能级别的波动。
- `DAILY`：中等杠杆，默认 5X，适配日线波段止损止盈。
- `H4`：执行级小仓位，默认 10X，但必须受 BOLL 开口、breakout 和严格止损约束。

周线仓的强平保护、名义价值上限和保证金占用必须按周线杠杆计算；不能因为系统默认 10X，就拒绝本应按周线低杠杆成立的周线仓。

周线仓交易执行必须绑定周线事件。即使系统在处理日线或 4H K 线时重新计算了周线状态，也只能用于展示或后续周线事件参考，不能在日线或 4H 事件上执行周线开仓、减仓或退出。周线仓的 `REDUCE_POSITION` / `EXIT_POSITION` 只能在周线收盘确认后触发。

周线仓减仓必须具备阶段记忆。同一笔周线仓中，趋势破坏、结构破坏、动能破坏各自最多触发一次减仓；已经触发过的破坏类型不能在后续周线重复减仓。若新的破坏类型出现，才允许进入下一次分批减仓。

## 6. 周线趋势状态

### 6.1 周线状态定义

```text
IF Weekly EMA15 > Weekly MA60:
    WEEKLY = BULL

IF Weekly EMA15 < Weekly MA60:
    WEEKLY = BEAR

IF Weekly EMA15 与 Weekly MA60 过近，且方向不稳定:
    WEEKLY = TRANSITION
```

后续需要量化：

- `EMA15 ≈ MA60` 的阈值。
- 是否要求周线收盘确认。
- 是否要求 EMA15 斜率同步。
- 是否加入 ADX / DI 动能确认。
- 是否加入周线结构高低点确认。

### 6.2 周线状态的作用

周线状态决定：

- 是否允许建立周线仓。
- 日线仓的方向偏好。
- 反向日线仓是否只能快进快出。
- 账户风险预算如何分配。
- 当前是否进入防御、减仓或清仓状态。

周线状态不应被 4H 波动频繁改变。

## 7. 周线仓规则

### 7.1 周线空仓开仓

当周线进入空头状态时，系统进入空头大环境。

开周线空仓的条件：

- Weekly EMA15 < Weekly MA60。
- Daily EMA15 < Daily MA60，形成日线死叉确认。
- 或者日线反弹后确认失败，重新转为死叉。
- 账户级风控允许。
- 强平安全距离允许。

此时建立的是周线空仓，不是日线空仓，也不是 4H 空仓。

周线空仓的入场价、初始防线和生命周期目标必须来自周线本身。初始趋势防线优先使用周线 MA60；周线结构高点只作为结构破坏减仓/退出观察，不作为默认初始硬止损。4H 只能作为执行观察或更低层级仓位信号，不能决定周线仓的止损、止盈或移动保护线。

仓位：

- 至少占本轮计划仓位预算的 60%。
- 若分批开仓，首次建仓和后续加仓的总和仍归属周线仓。
- 加仓必须重新检查总风险和强平安全距离。
- 周线仓必须使用周线独立杠杆和保证金模型，默认 2X；强平价、保证金占用、名义价值上限和开仓可行性都按周线杠杆计算。

### 7.2 周线仓不能强制持有到金叉

周线金叉是强制完全退出条件，但不是唯一退出或减仓条件。

周线空仓持有期间，如果趋势被逐步破坏，必须逐步减仓，而不是死扛到金叉。

周线仓退出分为三类：

- 趋势破坏减仓。
- 结构破坏减仓。
- 动能破坏减仓。

周线金叉只代表最终确认反转，触发剩余仓位全部退出。

日线金叉、日线反弹、4H 反弹、4H BOLL 收口、4H 普通止损或 4H 普通止盈，都不能直接平掉周线空仓。它们最多只能影响 `DAILY` 或 `H4` 自身仓位，或者作为周线动能破坏的辅助观察项；周线仓减仓或退出必须由周线 MA60、周线结构高点、周线动能破坏或周线金叉触发。

执行约束：日线或 4H 事件不能执行任何周线仓交易。周线减仓或退出即使依赖最新周线状态，也必须等到周线 K 线收盘事件由周线策略层发出。

阶段约束：趋势破坏、结构破坏、动能破坏必须写入周线仓生命周期状态。已写入的阶段不再重复触发；剩余仓位只在新的周线级破坏阶段或周线金叉/死叉时继续处理。

### 7.3 趋势破坏减仓

当价格重新站上周线 MA60：

- 不立即认定周线趋势完全反转。
- 先减掉部分周线仓。
- 这是趋势防线减仓，不是最终清仓。

建议初始规则：

```text
周线收盘站上 MA60:
    reduce WEEKLY position 30% - 50%
```

后续需要量化：

- 是否必须周线收盘确认。
- 是否要求连续 N 根周线站上。
- 是否允许极端突破提前减仓。

### 7.4 结构破坏减仓

结构破坏指周线空头结构不再保持，例如：

- 价格突破前一轮周线反弹高点。
- 周线形成更高低点和更高高点。
- 空头趋势线被有效突破。
- 周线关键压力位被收盘突破。

结构破坏不一定等于周线金叉，但必须触发减仓。

建议初始规则：

```text
周线结构高点被收盘突破:
    reduce WEEKLY position 30% - 50%
```

### 7.5 动能破坏减仓

动能破坏指趋势仍未金叉，但空头动能显著衰减。

可选信号：

- ADX 下降并低于阈值。
- DI- 优势消失。
- 周线连续收强反弹 K 线。
- 下跌创新低失败，反弹幅度扩大。

动能破坏应触发小比例减仓或停止加仓。

建议初始规则：

```text
空头动能衰减:
    stop adding WEEKLY
    reduce WEEKLY position 20% - 30%
```

### 7.6 周线金叉强制退出

当周线 EMA15 上穿周线 MA60，并完成金叉确认：

- 清掉剩余全部周线空仓。
- 停止开周线空环境下的日线顺势空单。
- 进入周线多头或转换期评估。

多头环境完全对称：

- 跌破周 MA60 是趋势防线减仓。
- 跌破关键周线结构是结构减仓。
- 多头动能衰减是动能减仓。
- 周线死叉确认后，强制退出剩余周线多仓。

## 8. 日线仓规则

### 8.1 日线仓互斥

同一交易对、同一周线环境下，日线仓只能存在一种：

```text
DAILY = REBOUND
或
DAILY = TREND
或
DAILY = NONE
```

不能同时持有日线反弹单和日线顺势单。

如果反弹单正在持有，系统不能再开顺势单。必须先按反弹单规则止盈、止损或确认反弹失败退出，再评估是否开顺势单。

### 8.2 周线空趋势下的日线反弹多单

当已经持有周线空仓，且日线转为金叉或形成明确反弹结构时，可以考虑日线反弹多单。

这个单属于日线仓，方向是多，模式是反弹。

特点：

- 只参与日线反弹。
- 必须快进快出。
- 不参与周线趋势反转判断。
- 不允许因反弹单盈利而切换为周线多头。

开仓前必须确认：

- 周线仍是空头大环境。
- 当前没有日线顺势空单。
- 日线确实金叉或形成反弹结构。
- 风险收益比足够。
- 反弹仓预算独立。

退出：

- 达到日线反弹止盈则平仓。
- 触发日线反弹止损则平仓。
- 日线反弹失败并重新转为空头结构时平仓。
- 账户级风控触发时强制退出。

### 8.3 周线空趋势下的顺势日线空单

当日线反弹单结束后，如果日线重新满足顺势做空条件，可以开顺势日线空单。

这个单属于日线仓，方向是空，模式是顺势。

开仓条件：

- 周线仍是空头大环境。
- 当前没有日线反弹多单。
- 日线反弹失败。
- 日线重新死叉，或跌破关键日线结构。
- 4H 执行条件允许。
- 不是追空入场。

仓位：

- 至少占本轮计划仓位预算的 30%。
- 若周线空仓仍在，顺势日线空单属于同向加风险。
- 开仓前必须重新计算总风险。

退出：

- 按日线空单自身规则止盈止损。
- 不影响周线仓剩余仓位。
- 除非触发账户级风控或周线级别反转。

### 8.4 周线多趋势下的日线仓

多头环境对称：

- 周线多趋势下，日线死叉可作为日线回调空单。
- 日线回调失败并重新金叉后，可开顺势日线多单。
- 日线反向单和顺势单互斥，不能同时存在。
- 反向日线仓只能是战术仓，不能升级为周线仓。

## 9. 4H 仓规则

4H 的使命是减少低质量入场，同时捕捉趋势初期 breakout。

由于 4H 趋势波段容易震荡，且价格可能在很窄的 Bollinger Bands 区间内反复波动，所以 4H 条件必须严格。

### 9.1 4H 禁止交易状态

```text
IF 4H BOLL width contracting:
    H4 = NO_TRADE
```

禁止事项：

- 不开 4H 仓。
- 不因 4H 小波动频繁止盈止损。
- 不因短线假突破打乱周线和日线仓位。

### 9.2 4H breakout 模式

必须加入 breakout 模式，否则容易错过趋势初期。

4H breakout 用于捕捉从压缩区间向趋势段切换的第一段机会。

允许条件：

- 4H BOLL 从收缩转为开口。
- 价格收盘突破最近结构高点或低点。
- 突破方向与周线或日线允许方向一致。
- 突破 K 线不能距离结构位过远，避免追单。
- 风险收益比足够。

示例：

```text
IF 4H BOLL opens
AND 4H close breaks structure
AND direction aligns with allowed bias:
    allow H4 breakout entry
```

breakout 仓属于 4H 仓，预算最多 10%。它不能替代周线仓，也不能与日线仓混淆。

### 9.3 4H pullback 模式

如果错过 breakout，允许等待回踩后再入场。

允许条件：

- 4H 已经完成有效突破。
- 价格回踩突破位、均线或结构区。
- 回踩不跌回原震荡区。
- 出现顺势确认 K 线。

示例：

```text
IF breakout confirmed
AND price retests breakout zone
AND retest holds:
    allow H4 pullback entry
```

### 9.4 4H continuation 模式

趋势已经运行时，可以做 continuation，但必须更严格。

允许条件：

- BOLL 仍开口。
- 趋势结构仍延续。
- 未明显远离均线或结构位。
- 不是连续大幅上涨后的追多，或连续大幅下跌后的追空。

### 9.5 4H 仓原则

4H 仓最多占本轮计划仓位预算的 10%。

原则：

- BOLL 不开口，不开 4H 仓。
- 结构不确认，不开 4H 仓。
- breakout 可以开，但必须防追单。
- pullback 可以开，但必须确认回踩有效。
- continuation 可以开，但必须限制过度延伸。
- 4H 仓不允许频繁补仓摊平。

## 10. 统一风险预算

基础预算：

```text
TOTAL_POSITION_BUDGET = 100%

WEEKLY = >= 60%
DAILY  = >= 30%
H4     = <= 10%
```

该预算表示策略层级分配，不等于账户资金直接使用比例。

落地时必须同时计算：

- 单笔计划亏损。
- 总计划亏损。
- 保证金占用。
- 名义价值。
- 杠杆倍数。
- 强平价距离。
- 同向仓位总风险。
- 反向仓位净敞口。
- 手续费与资金费率成本。

### 10.1 同向风险

周线空仓 + 顺势日线空仓属于同向风险。

开顺势日线仓前，必须确认：

- 周线仓仍符合持有条件。
- 日线顺势信号不是追单。
- 同向总计划风险没有超预算。
- 强平安全距离仍然足够。

### 10.2 反向风险

周线空仓 + 日线反弹多仓属于反向战术仓。

反向仓不能被理解成无风险对冲。它可能降低净敞口，也可能增加手续费、滑点和波动损耗。

开反向仓前，必须确认：

- 它是日线战术仓，不是周线反转仓。
- 当前没有日线顺势仓。
- 风险预算独立。
- 持仓周期更短。
- 退出条件更严格。

### 10.3 仓位预算不足时的取舍

当总预算不足时，优先级为：

```text
周线仓
> 日线顺势仓
> 日线反弹仓
> 4H 仓
```

系统不应为了低级别机会，挤占周线仓的安全空间。

---

## 11. 冲突处理

### 11.1 周线空仓 + 日线金叉

含义：

- 周线空头大环境仍在。
- 日线出现反弹。
- 可考虑日线反弹多单。

处理：

- 不平掉全部周线空仓。
- 可保留周线空仓。
- 不允许同时持有日线顺势空单。
- 若反弹强到价格站上周 MA60，则触发周线趋势防线减仓。

### 11.2 周线空仓 + 日线反弹失败

含义：

- 周线空头大环境继续。
- 日线反弹结束。
- 顺势空头机会恢复。

处理：

- 先平掉日线反弹多单。
- 再评估是否开日线顺势空单。
- 周线空仓继续按周线规则管理。

### 11.3 周线空仓 + 4H 窄幅震荡

含义：

- 大趋势仍在，但短线执行质量差。

处理：

- 不开 4H 仓。
- 不因 4H 小波动频繁处理周线仓。
- 等待 BOLL 开口、breakout 或结构突破。

### 11.4 周线空仓 + 周线趋势破坏

含义：

- 周线空头仍未确认反转，但趋势质量下降。

处理：

- 根据趋势破坏、结构破坏、动能破坏逐步减仓。
- 停止继续加周线空仓。
- 降低或禁止同向日线顺势空单。

### 11.5 周线空仓 + 周线金叉

含义：

- 周线级别趋势反转确认。

处理：

- 强制清掉剩余全部周线空仓。
- 停止开周线空环境下的日线空单。
- 重新进入周线多头或转换期评估。

多头环境完全对称。

---

## 12. 状态切换优先级

交易系统必须有清晰优先级：

```text
账户级风控
> 强平风险
> 硬止损
> Equity Guard 资金曲线保护
> 周线金叉/死叉强制退出
> 周线趋势破坏减仓
> 周线结构破坏减仓
> 周线动能破坏减仓
> Lifecycle 仓位生命周期推进
> Throttle 交易节流
> 日线仓止损止盈
> 日线反弹/顺势互斥切换
> 4H 禁止交易状态
> 新开仓
```

解释：

- 风控永远优先于策略。
- 已有仓位管理优先于新开仓。
- 周线仓管理优先于日线仓。
- 日线仓互斥规则优先于新开日线单。
- 4H 仓只在严格条件下参与。

---

## 13. 控制层

控制层不新增仓位类型。它只决定当前信号是否允许执行、是否需要降仓、是否进入冷却、仓位是否进入保护阶段，以及账户是否必须停止新开仓。

本阶段写入 5 个控制模块：

```text
Regime Tagging = 市场状态识别
Throttle       = 交易节流
Signal Score   = 信号评分
Lifecycle      = 仓位生命周期管理
Equity Guard   = 资金曲线保护
```

### 13.1 市场状态识别 Regime Tagging

Regime Tagging 用于识别当前市场更像趋势、震荡、转换、突破初期，还是趋势衰竭。

建议状态：

```text
TREND       = 趋势
RANGE       = 震荡
TRANSITION  = 转换
BREAKOUT    = 突破初期
EXHAUSTION  = 趋势衰竭
UNKNOWN     = 数据不足或状态不明
```

Regime Tagging 的作用：

- `TREND`：允许周线仓、日线顺势仓和 4H continuation。
- `RANGE`：禁止追单，限制 4H 仓，日线反弹仓也必须快进快出。
- `TRANSITION`：降低仓位或禁止新开仓。
- `BREAKOUT`：允许 4H breakout，但必须防追单和防假突破。
- `EXHAUSTION`：禁止加仓，已有盈利仓进入保护或减仓。
- `UNKNOWN`：默认不开新仓。

Regime Tagging 不能直接决定买卖，只能影响允许、禁止、降仓和保护强度。

### 13.2 交易节流 Throttle

Throttle 用于防止过度交易，尤其防止震荡行情里反复开平、止损后立刻反手、4H breakout 假突破后连续重试。

建议节流规则：

- 同一交易对、同一层级，平仓后必须等待冷却期。
- 同一方向连续止损 N 次后，暂停该方向新开仓。
- 日线反弹单止损后，不能立刻开新的日线反弹单。
- 4H breakout 失败后，必须等待新的结构或新的 BOLL 开口。
- 周线趋势破坏减仓后，禁止立刻重新加回周线仓。

Throttle 不改变趋势判断，只控制交易频率。

### 13.3 信号评分 Signal Score

Signal Score 用于区分强信号、普通信号和弱信号。

评分只做辅助，不允许覆盖硬风控。

建议分数区间：

```text
score >= 80: 强信号，可按标准预算执行
60 <= score < 80: 普通信号，可执行但不加仓
40 <= score < 60: 弱信号，降仓或等待
score < 40: 拒绝开仓
```

评分维度可以包括：

- 周线方向一致性。
- 日线结构确认程度。
- 4H breakout / pullback 质量。
- BOLL 开口质量。
- 入场点是否远离结构位。
- 风险收益比。
- 当前 Regime 是否匹配。
- 是否刚经历连续亏损或冷却期。

Signal Score 不能让系统无视止损、强平、Equity Guard、Throttle 或仓位互斥规则。

### 13.4 仓位生命周期 Lifecycle

Lifecycle 用于管理仓位从开仓到退出的演化，不让盈利仓长期裸奔，也不让亏损仓被动拖延。

建议生命周期：

```text
OPENED         = 刚开仓
CONFIRMED      = 行情按预期运行
PROTECTED      = 止损已推进，风险降低
PROFIT_LOCKED  = 已锁定部分利润
REDUCED        = 已按规则减仓
EXITED         = 已退出
```

Lifecycle 的作用：

- 新开仓后先进入 `OPENED`。
- 价格按预期运行后进入 `CONFIRMED`。
- 达到一定 R 倍数或结构推进后进入 `PROTECTED`。
- 盈利达到目标后进入 `PROFIT_LOCKED`。
- 趋势、结构或动能破坏后进入 `REDUCED`。
- 触发止损、强制退出或最终目标后进入 `EXITED`。

不同层级仓位使用不同生命周期节奏：

- 周线仓更重视趋势、结构和动能演化。
- 日线仓更重视快进快出和反弹失败。
- 4H 仓更重视 breakout 是否延续、是否假突破。

### 13.5 资金曲线保护 Equity Guard

Equity Guard 是账户级保护。它优先于所有策略信号。

建议保护规则：

- 单日最大亏损达到阈值：停止新开仓。
- 单周最大亏损达到阈值：停止新开仓并降低已有风险。
- 连续亏损 N 笔：进入冷却状态。
- 从权益新高回撤达到阈值：整体降仓。
- 当前回撤超过最大允许值：只允许减仓和平仓。

Equity Guard 触发后：

- 禁止新开仓。
- 禁止加仓。
- 可允许保护性减仓。
- 可允许已有仓位按止盈止损退出。
- 必要时强制降低总风险。

---

## 14. 禁止交易状态

以下状态默认不允许新开仓：

- 周线处于 TRANSITION，方向不稳定。
- 日线和周线方向冲突，但没有明确反弹仓规则。
- 已有日线反弹单时，又出现日线顺势单信号。
- 已有日线顺势单时，又出现日线反弹单信号。
- 4H BOLL 未开口，且没有有效 breakout。
- Regime Tagging = RANGE，且信号属于追单。
- Regime Tagging = UNKNOWN。
- Signal Score 低于最低开仓阈值。
- Throttle 冷却期未结束。
- Equity Guard 已触发暂停交易。
- 入场距离均线或结构位过远，属于追单。
- 止损距离过大，导致仓位过小或强平距离不足。
- 手续费、滑点、资金费率吞噬计划风险。
- 数据不足、指标不完整或 K 线未收盘。
- 账户达到最大回撤、连续亏损或风控熔断。

---

## 15. 需要量化的规则

### 15.1 日线死叉

需要明确：

- 是否要求 Daily EMA15 收盘低于 Daily MA60。
- 是否要求连续 N 根日线确认。
- 是否要求 EMA15 斜率向下。
- 是否要求价格也在 MA60 下方。

### 15.2 日线反弹失败

可选定义：

- 日线金叉后重新死叉。
- 日线反弹未突破前高，并跌破最近 swing low。
- 日线收盘重新跌破 MA60。
- 反弹期间动能无法延续，随后空头结构恢复。

### 15.3 周线趋势破坏

需要明确：

- 周线收盘站上或跌破 MA60 是否触发。
- 是否需要连续 N 根周线确认。
- 是否按 30%、50% 或其他比例减仓。
- 是否允许强突破提前减仓。

### 15.4 周线结构破坏

需要明确：

- 使用哪个 swing high / swing low。
- 是收盘突破还是影线突破。
- 是否需要突破后确认。
- 结构破坏后减仓比例是多少。

### 15.5 周线动能破坏

需要明确：

- ADX 低于什么阈值算动能衰减。
- DI 优势消失是否触发。
- 连续几根周线反弹算动能破坏。
- 动能破坏是停止加仓还是直接减仓。

### 15.6 周线金叉确认

需要明确：

- EMA15 上穿 MA60 是否必须发生在周线收盘后。
- 是否要求连续 N 根周线保持金叉。
- 是否要求价格也站稳 MA60。
- 是否要求周线结构高点突破。

### 15.7 BOLL 开口

需要明确：

- 使用 4H Bollinger Bands width。
- width 是否高于过去 N 根均值。
- width 是否连续扩大。
- 是否要求价格突破上轨/下轨或突破结构。

### 15.8 4H breakout

需要明确：

- 突破最近多少根 4H 结构。
- 收盘突破还是影线突破。
- 突破后最大允许偏离结构位多少 ATR。
- 是否需要成交量或波动扩张确认。
- false breakout 如何止损。

### 15.9 快进快出

日线反弹仓的“快进快出”必须量化：

- 最大持仓 K 线数量。
- 最大亏损比例。
- 最小风险收益比。
- 是否触发移动止损。
- 反弹失败时是否立即退出。

### 15.10 控制层参数

需要明确：

- Throttle 冷却期按多少根 K 线或多少时间计算。
- 连续亏损几次触发暂停。
- Regime Tagging 的 TREND / RANGE / BREAKOUT / EXHAUSTION 如何判定。
- Signal Score 每个维度权重是多少。
- Lifecycle 每个状态如何推进。
- Equity Guard 的日亏损、周亏损、权益回撤阈值是多少。

---

## 16. 后续细化顺序

建议按以下顺序继续完善：

1. 先量化周线状态：BULL / BEAR / TRANSITION。
2. 再量化周线仓开仓、趋势破坏、结构破坏、动能破坏和金叉强制退出。
3. 再量化日线反弹单和顺势单，并实现互斥状态。
4. 再量化 4H breakout、pullback、continuation。
5. 加入 Regime Tagging，用市场状态决定允许、禁止和降仓。
6. 加入 Throttle，限制连续交易和假突破重试。
7. 加入 Lifecycle，管理盈利保护、减仓和退出。
8. 加入 Equity Guard，保护账户资金曲线。
9. 最后加入 Signal Score，用于过滤弱信号和控制仓位强弱。
10. 把仓位预算映射为真实风险、保证金、杠杆和强平距离。

每一步都必须能单独回测，不能一次性把所有规则混在一起改。

---

## 17. 核心原则

- 仓位只分周线、日线、4H 三类。
- 周线是大环境，也是核心仓位层。
- 周线金叉或死叉是强制完全退出条件。
- 周线趋势破坏时必须逐步减仓，不能死扛。
- 日线反弹单和顺势单不能同时存在。
- 4H 必须支持 breakout，否则会错过趋势初期。
- 4H 仍必须严格过滤，避免震荡中频繁交易。
- Regime Tagging 只决定市场状态，不直接决定买卖。
- Throttle 控制交易频率，避免过度交易。
- Signal Score 只能辅助过滤和降仓，不能覆盖硬风控。
- Lifecycle 管理仓位演化，避免盈利仓裸奔或亏损仓拖延。
- Equity Guard 是账户级最高防线。
- 仓位比例必须落到统一风险预算。
- 任何不确定状态，默认不开新仓。

## 18. 一句话定义

```text
A multi-timeframe position hierarchy:

Weekly = regime + core position + staged exits
Daily  = mutually exclusive rebound or trend trade
H4     = strict execution with breakout support
Control = regime, throttle, score, lifecycle, equity guard
Risk    = unified budget and defense line
```
