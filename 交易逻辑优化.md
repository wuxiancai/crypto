# Multi-Timeframe Trading System

## 1. 文档目的

本文档用于明确交易系统的大防线和仓位层级。它先定义“什么仓位在什么环境下可以存在”，再逐步补充入场、止损、止盈和风险约束。

这不是最终参数文档。后续所有“死叉、反弹失败、BOLL 开口、结构突破、站上周 MA60”等词，都必须继续量化，才能进入回测或自动执行。

核心目标：

- 周线决定大环境，也对应最高级别的核心仓位。
- 日线在周线环境下决定战术仓位。
- 4H 只负责严格执行和过滤低质量交易。
- 风险预算统一管理，仓位不能简单叠加。

---

## 2. 核心定义

### 2.1 周线不是单纯过滤器

周线既是趋势状态，也是核心仓位层。

当周线进入空头或多头状态，并获得日线确认时，系统可以建立周线核心仓。这个仓位计划长线持有，不能被日线或 4H 的短期波动随意改变身份。

### 2.2 日线是周线环境内的战术层

日线不能直接推翻周线趋势。日线只能在周线大环境下产生两类仓位：

- 反向战术仓：例如周线空趋势下的日线反弹多单。
- 顺势战术仓：例如周线空趋势下的顺势日线空单。

### 2.3 4H 是执行层

4H 只负责提高入场质量。它不决定大趋势，不决定核心仓位方向，也不应该制造频繁交易。

若 4H 处于窄幅震荡、BOLL 未开口、结构不清晰，系统宁可不开 4H 仓，也不为了交易而交易。

### 2.4 仓位比例不是账户资金比例

本文中的 60%、30%、10% 是计划仓位预算的层级分配，不等于直接使用账户资金比例，也不等于保证金比例。

落地时必须转换为：

- 单笔计划亏损。
- 总计划亏损。
- 保证金占用。
- 名义价值。
- 杠杆倍数。
- 强平安全距离。

---

## 3. 总体分层

```text
Weekly = 大趋势状态 + 周线核心仓
Daily  = 周线环境下的战术仓
H4     = 入场执行器 + 低级别机会仓
Risk   = 统一预算与防线
```

周线是第一防线。它决定系统处于牛市、熊市还是转换期，并决定是否允许建立周线级别核心仓。

日线是第二防线。它负责确认周线仓位是否可以开仓，也负责在周线趋势内识别反弹和顺势恢复。

4H 是第三防线。它负责执行，但只有当波动结构展开、BOLL 开口、突破确认时才允许参与。

## 4. 仓位身份表

| 仓位类型 | 大环境 | 方向 | 仓位身份 | 预算层级 | 持仓目标 |
| --- | --- | --- | --- | --- | --- |
| `WEEKLY_SHORT_CORE` | 周线空头 | 空 | 周线核心仓 | >= 60% | 长线持有 |
| `WEEKLY_LONG_CORE` | 周线多头 | 多 | 周线核心仓 | >= 60% | 长线持有 |
| `DAILY_REBOUND_LONG_UNDER_WEEKLY_BEAR` | 周线空头 | 多 | 日线反弹仓 | 独立战术预算 | 快进快出 |
| `DAILY_REBOUND_SHORT_UNDER_WEEKLY_BULL` | 周线多头 | 空 | 日线回调仓 | 独立战术预算 | 快进快出 |
| `DAILY_TREND_SHORT_UNDER_WEEKLY_BEAR` | 周线空头 | 空 | 顺势日线仓 | >= 30% | 趋势延续 |
| `DAILY_TREND_LONG_UNDER_WEEKLY_BULL` | 周线多头 | 多 | 顺势日线仓 | >= 30% | 趋势延续 |
| `H4_EXECUTION_TRADE` | 任一明确环境 | 顺势优先 | 4H 执行仓 | <= 10% | 短线执行 |

仓位身份一旦建立，后续止盈止损必须按该仓位身份执行。不能因为价格短期波动，把周线仓当日线仓处理，也不能把日线反弹仓升级为周线核心仓。

## 5. 周线趋势状态

### 5.1 周线状态定义

```text
IF Weekly EMA15 > Weekly MA60:
    WEEKLY = BULL

IF Weekly EMA15 < Weekly MA60:
    WEEKLY = BEAR

IF Weekly EMA15 与 Weekly MA60 过近，且方向不稳定:
    WEEKLY = TRANSITION
```

后续需要量化：

- `EMA15 ≈ MA60` 的阈值。
- 是否要求周线收盘确认。
- 是否要求 EMA15 斜率同步。
- 是否加入 ADX / DI 动能确认。
- 是否加入周线结构高低点确认。

### 5.2 周线状态的作用

周线状态决定：

- 是否允许建立周线核心仓。
- 日线战术仓的方向偏好。
- 反向日线仓是否只能快进快出。
- 账户风险预算如何分配。
- 当前是否进入防御、减仓或清仓状态。

周线状态不应被 4H 波动频繁改变。

---

## 6. 周线核心仓

### 6.1 周线空头核心仓开仓

当周线进入空头状态时，系统进入空头大环境。

开 `WEEKLY_SHORT_CORE` 的条件：

- Weekly EMA15 < Weekly MA60。
- Daily EMA15 < Daily MA60，形成日线死叉确认。
- 或者日线反弹后确认失败，重新转为死叉。
- 账户级风控允许。
- 强平安全距离允许。

此时建立的是周线空单，不是日线空单，也不是 4H 空单。它的持仓目标是吃周线级别趋势。

仓位：

- 至少占本轮计划仓位预算的 60%。
- 若分批开仓，首次建仓和后续加仓的总和仍归属周线核心仓。
- 加仓必须重新检查总风险和强平安全距离。

执行：

- 日线负责确认是否具备开周线仓的环境。
- 4H 可以帮助寻找更优入场点。
- 4H 不能改变这个仓位的周线身份。

### 6.2 周线空头核心仓减仓

当价格重新站上周线 MA60：

- 不立即认定周线趋势反转。
- 先减掉周线空单的一半仓位。
- 这是“周线防线减仓”，不是最终清仓。

建议后续定义：

- 是否要求周线收盘站上 MA60。
- 是否要求连续 N 根周线确认。
- 是否允许周内强突破提前触发防御减仓。

### 6.3 周线空头核心仓清仓

剩余周线空单继续持有，直到周线金叉确认。

清仓条件：

- Weekly EMA15 上穿 Weekly MA60。
- 周线金叉被确认。
- 或账户级风控要求强制退出。

### 6.4 周线多头核心仓

多头逻辑完全对称：

- Weekly EMA15 > Weekly MA60。
- Daily EMA15 > Daily MA60，或日线回调失败后重新金叉。
- 建立 `WEEKLY_LONG_CORE`。
- 价格跌破周线 MA60 时先减半。
- 周线死叉确认时清仓。

## 7. 周线趋势下的日线战术仓

日线仓必须依附于周线大环境。日线可以做顺势，也可以做反弹，但两者目标完全不同。

### 7.1 周线空趋势下的日线反弹多单

当已经持有周线空单，且日线转为金叉时，可以根据规则开日线反弹多单。

```text
POSITION_TYPE = DAILY_REBOUND_LONG_UNDER_WEEKLY_BEAR
```

这个单是周线空趋势下的日线反弹仓，不是周线多单。

特点：

- 只参与日线反弹。
- 必须快进快出。
- 不参与周线趋势反转判断。
- 不允许因反弹多单盈利而切换为周线多头。

开仓前必须确认：

- 周线仍是空头大环境。
- 日线确实金叉或形成反弹结构。
- 风险收益比足够。
- 反弹仓预算独立。

退出：

- 达到日线反弹止盈则平仓。
- 触发日线反弹止损则平仓。
- 日线反弹失败并重新转为空头结构时平仓。
- 账户级风控触发时强制退出。

### 7.2 周线空趋势下的顺势日线空单

当日线反弹多单结束后，如果日线重新满足顺势做空条件，可以开顺势日线空单。

```text
POSITION_TYPE = DAILY_TREND_SHORT_UNDER_WEEKLY_BEAR
```

开仓条件：

- 周线仍是空头大环境。
- 日线反弹失败。
- 日线重新死叉，或跌破关键日线结构。
- 4H 执行条件允许。
- 不是追空入场。

仓位：

- 至少占本轮计划仓位预算的 30%。
- 若周线核心空单仍在，顺势日线空单属于同向加风险。
- 开仓前必须重新计算总风险。

退出：

- 按日线空单自身规则止盈止损。
- 不影响周线核心空单的剩余仓位。
- 除非触发账户级风控或周线级别反转。

### 7.3 周线多趋势下的日线战术仓

多头环境对称：

- 周线多趋势下，日线死叉可作为日线回调空单。
- 日线回调失败并重新金叉后，可开顺势日线多单。
- 反向日线仓只能是战术仓，不能升级为周线核心仓。

## 8. 4H 执行层

4H 的使命是减少低质量入场，不是增加交易次数。

由于 4H 趋势波段容易震荡，且价格可能在很窄的 Bollinger Bands 区间内反复波动，所以 4H 条件必须更严格。

### 8.1 4H 禁止交易状态

```text
IF 4H BOLL width contracting:
    H4 = NO_TRADE
```

禁止事项：

- 不开独立 4H 仓。
- 不因 4H 小波动频繁止盈止损。
- 不因短线假突破打乱周线和日线仓位。

### 8.2 4H 允许执行状态

```text
IF 4H BOLL width expanding
AND 4H structure break confirmed:
    H4 = ENTRY_CONFIRM
```

允许事项：

- 为周线核心仓寻找更好的入场价格。
- 为日线顺势仓寻找执行点。
- 在极严格条件下允许小比例 4H 执行仓。

### 8.3 4H 独立仓位

若保留 4H 独立仓，应定义为：

```text
POSITION_TYPE = H4_EXECUTION_TRADE
```

仓位：

- 最多占本轮计划仓位预算的 10%。
- 只在 BOLL 开口、结构突破、风险收益比足够时开仓。

原则：

- BOLL 不开口，不开 4H 仓。
- 结构不确认，不开 4H 仓。
- 风险收益比不足，不开 4H 仓。
- 4H 仓不允许频繁补仓摊平。

## 9. 统一风险预算

基础预算：

```text
TOTAL_POSITION_BUDGET = 100%

WEEKLY_CORE = >= 60%
DAILY_TREND = >= 30%
H4_EXECUTION = <= 10%
```

该预算表示策略层级分配，不等于账户资金直接使用比例。

落地时必须同时计算：

- 单笔计划亏损。
- 总计划亏损。
- 保证金占用。
- 名义价值。
- 杠杆倍数。
- 强平价距离。
- 同向仓位总风险。
- 反向仓位净敞口。
- 手续费与资金费率成本。

### 9.1 同向风险

周线空单 + 顺势日线空单属于同向风险。

开顺势日线空单前，必须确认：

- 周线核心仓仍符合持有条件。
- 日线顺势信号不是追空。
- 同向总计划风险没有超预算。
- 强平安全距离仍然足够。

### 9.2 反向风险

周线空单 + 日线反弹多单属于反向战术仓。

反向仓不能被理解成无风险对冲。它可能降低净敞口，也可能增加手续费、滑点和波动损耗。

开反向仓前，必须确认：

- 它是日线战术仓，不是周线反转仓。
- 风险预算独立。
- 持仓周期更短。
- 退出条件更严格。

### 9.3 仓位预算不足时的取舍

当总预算不足时，优先级为：

```text
周线核心仓
> 顺势日线仓
> 反向日线战术仓
> 4H 独立执行仓
```

系统不应为了低级别机会，挤占周线核心仓的安全空间。

## 10. 冲突处理

### 10.1 周线空单 + 日线金叉

含义：

- 周线空头大环境仍在。
- 日线出现反弹。
- 可考虑日线反弹多单。

处理：

- 不平掉全部周线空单。
- 可保留周线核心空单。
- 若反弹强到价格站上周 MA60，则触发周线防线减仓。

### 10.2 周线空单 + 日线反弹失败

含义：

- 周线空头大环境继续。
- 日线反弹结束。
- 顺势空头机会恢复。

处理：

- 平掉日线反弹多单。
- 若条件满足，可开顺势日线空单。
- 周线核心空单继续按周线规则管理。

### 10.3 周线空单 + 4H 窄幅震荡

含义：

- 大趋势仍在，但短线执行质量差。

处理：

- 不开 4H 独立仓。
- 不因 4H 小波动频繁处理周线仓。
- 等待 BOLL 开口和结构突破。

### 10.4 周线空单 + 周线金叉

含义：

- 周线级别趋势反转确认。

处理：

- 清掉剩余周线空单。
- 停止开周线空趋势下的日线空单。
- 重新进入周线多头或转换期评估。

多头环境完全对称。

## 11. 状态切换优先级

交易系统必须有清晰优先级：

```text
账户级风控
> 强平风险
> 硬止损
> 周线核心趋势反转
> 周线 MA60 防线减仓
> 日线战术仓止损止盈
> 4H 禁止交易状态
> 新开仓
```

解释：

- 风控永远优先于策略。
- 已有仓位管理优先于新开仓。
- 周线核心仓管理优先于日线战术仓。
- 日线战术仓优先于 4H 机会仓。

---

## 12. 禁止交易状态

以下状态默认不允许新开仓：

- 周线处于 TRANSITION，方向不稳定。
- 日线和周线方向冲突，但没有明确反弹仓规则。
- 4H BOLL 未开口，且只存在短线震荡信号。
- 入场距离均线或结构位过远，属于追单。
- 止损距离过大，导致仓位过小或强平距离不足。
- 手续费、滑点、资金费率吞噬计划风险。
- 数据不足、指标不完整或 K 线未收盘。
- 账户达到最大回撤、连续亏损或风控熔断。

## 13. 需要量化的规则

### 13.1 日线死叉

需要明确：

- 是否要求 Daily EMA15 收盘低于 Daily MA60。
- 是否要求连续 N 根日线确认。
- 是否要求 EMA15 斜率向下。
- 是否要求价格也在 MA60 下方。

### 13.2 日线反弹失败

可选定义：

- 日线金叉后重新死叉。
- 日线反弹未突破前高，并跌破最近 swing low。
- 日线收盘重新跌破 MA60。
- 反弹期间动能无法延续，随后空头结构恢复。

### 13.3 周线站上 MA60

建议定义为：

- 周线收盘价站上 Weekly MA60。
- 或连续 N 根周线收盘在 Weekly MA60 上方。

不建议用周内价格刺破直接触发核心仓减半。

### 13.4 周线金叉确认

需要明确：

- EMA15 上穿 MA60 是否必须发生在周线收盘后。
- 是否要求连续 N 根周线保持金叉。
- 是否要求价格也站稳 MA60。
- 是否要求周线结构高点突破。

### 13.5 BOLL 开口

需要明确：

- 使用 4H Bollinger Bands width。
- width 是否高于过去 N 根均值。
- width 是否连续扩大。
- 是否要求价格突破上轨/下轨或突破结构。

### 13.6 结构突破确认

需要明确：

- swing high / swing low 的 lookback。
- 是收盘突破还是影线突破。
- 是否需要回踩确认。
- 突破后允许追单的最大距离。

### 13.7 快进快出

日线反弹仓的“快进快出”必须量化：

- 最大持仓 K 线数量。
- 最大亏损比例。
- 最小风险收益比。
- 是否触发移动止损。
- 反弹失败时是否立即退出。

---

## 14. 后续细化顺序

建议按以下顺序继续完善：

1. 先量化周线状态：BULL / BEAR / TRANSITION。
2. 再量化周线核心仓开仓、减半、清仓。
3. 再量化日线反弹仓和顺势仓。
4. 再量化 4H BOLL 开口和结构突破。
5. 最后把仓位预算映射为真实风险、保证金、杠杆和强平距离。

每一步都必须能单独回测，不能一次性把所有规则混在一起改。

---

## 15. 核心原则

- 周线是大环境，也是核心仓位层。
- 日线是战术层，只在周线环境中解释。
- 4H 是执行层，宁可错过，也不在窄幅震荡中频繁交易。
- 周线仓、日线仓、4H 仓必须有不同身份。
- 顺势仓和反向仓必须分开管理。
- 仓位比例必须落到统一风险预算。
- 任何不确定状态，默认不开新仓。

## 16. 一句话定义

```text
A multi-timeframe position hierarchy:

Weekly = regime + core position
Daily  = tactical position inside weekly regime
H4     = strict execution filter
Risk   = unified budget and defense line
```
