# Multi-Timeframe Trading System

## 1. 文档目的

本文档用于明确交易系统的大防线，而不是一次性写完所有细节参数。

核心目标：

- 周线决定大环境，也对应最高级别的周线仓位。
- 日线在周线环境内决定战术交易方向。
- 4H 只负责严格执行，不负责改变大方向。
- 所有仓位都必须受统一风险预算约束。

这里的“仓位比例”默认指本轮策略计划仓位预算的分配比例。后续落地时，必须进一步映射为风险金额、保证金、名义价值和强平安全距离，不能理解为无风控满仓。

---

## 2. 总体分层

系统分为三层：

```text
Weekly = 大趋势状态 + 周线核心仓
Daily  = 周线环境下的战术仓
H4     = 入场执行器 + 低级别机会仓
Risk   = 统一预算与防线
```

周线是第一防线。周线方向不只是过滤条件，而是决定系统当前处于什么大环境，以及是否允许建立周线级别核心仓。

日线是第二防线。日线不推翻周线大环境，但可以在周线趋势中识别反弹、多空切换和顺势恢复。

4H 是第三防线。4H 不负责判断长期方向，只负责在条件足够清晰时执行入场。若 4H 处于窄幅震荡或 Bollinger Bands 未开口，宁可不开仓。

---

## 3. 周线趋势状态

### 3.1 周线状态定义

```text
IF Weekly EMA15 > Weekly MA60:
    WEEKLY = BULL

IF Weekly EMA15 < Weekly MA60:
    WEEKLY = BEAR

IF Weekly EMA15 与 Weekly MA60 过近，且方向不稳定:
    WEEKLY = TRANSITION
```

后续需要补充：

- `EMA15 ≈ MA60` 的量化阈值。
- 是否要求周线收盘确认。
- 是否要求 EMA15 斜率同步。
- 是否加入 ADX / DI 或结构高低点确认。

### 3.2 周线状态的意义

周线状态决定：

- 是否允许建立周线核心仓。
- 日线战术仓的方向偏好。
- 反向日线仓是否只能作为短线反弹仓。
- 账户总风险预算如何分配。

周线状态不应该被 4H 波动频繁改变。

---

## 4. 周线核心仓

### 4.1 周线空头核心仓

当周线进入空头状态时，系统进入空头大环境。

开周线空单的条件：

- Weekly EMA15 < Weekly MA60。
- 并且 Daily EMA15 < Daily MA60，形成日线死叉确认。
- 或者日线反弹后确认失败，重新转为死叉。

此时建立的仓位是：

```text
POSITION_TYPE = WEEKLY_SHORT_CORE
```

它不是日线空单，也不是 4H 空单，而是计划长线持有的周线空单。

仓位：

- 至少占本轮计划仓位预算的 60%。
- 若继续加仓，必须仍受总预算、总风险和强平安全距离约束。

执行：

- 日线负责确认是否具备开周线仓的环境。
- 4H 可以用于寻找更好的入场价格，但不能改变仓位身份。

### 4.2 周线空头核心仓减仓和平仓

当价格重新站上周线 MA60：

- 不立即认定周线趋势完全反转。
- 先减掉周线空单的一半仓位。
- 这更准确地说是“趋势防线减仓”或“保护性减仓”，不是最终趋势平仓。

剩余仓位：

- 继续按周线空头核心仓管理。
- 只有当周线 EMA15 上穿周线 MA60，形成周线金叉确认时，才全部平仓。

建议后续补充：

- “站上周线 MA60”应以周线收盘确认，避免周内假突破。
- “周线金叉确认”是否需要连续 1-2 根周线收盘确认。
- 若价格快速突破周 MA60 且波动异常，是否触发额外风控退出。

### 4.3 周线多头核心仓

多头逻辑应与空头完全对称：

- Weekly EMA15 > Weekly MA60。
- Daily EMA15 > Daily MA60，或日线回调失败后重新金叉。
- 建立 `WEEKLY_LONG_CORE`。
- 价格跌破周线 MA60 时先减半。
- 周线死叉确认时清仓。

## 5. 周线趋势下的日线战术仓

日线仓必须依附于周线大环境。日线可以做顺势，也可以做反弹，但二者的仓位身份、持仓目标和退出规则不同。

### 5.1 周线空趋势下的日线反弹多单

当已经持有周线空单，且日线转为金叉时，可以根据规则开日线反弹多单：

```text
POSITION_TYPE = DAILY_REBOUND_LONG_UNDER_WEEKLY_BEAR
```

这个单是日线反弹仓，不是周线多单。

特点：

- 只参与周线空趋势中的日线反弹。
- 必须快进快出。
- 不参与周线趋势反转判断。
- 不允许因为反弹多单盈利就把系统切换为周线多头。

仓位：

- 使用独立日线反弹预算。
- 建议低于顺势日线仓。
- 后续需明确最大风险、最大持仓周期和最大回撤。

退出：

- 触发日线反弹止盈则平仓。
- 触发日线反弹止损则平仓。
- 若日线反弹失败并重新转为空头结构，应平掉反弹多单。

### 5.2 周线空趋势下的顺势日线空单

当日线反弹多单结束后，如果日线重新满足顺势做空条件，可以开顺势日线空单：

```text
POSITION_TYPE = DAILY_TREND_SHORT_UNDER_WEEKLY_BEAR
```

开仓条件：

- 周线仍是空头大环境。
- 日线反弹失败。
- 日线重新死叉，或跌破关键日线结构。
- 4H 执行条件允许。

仓位：

- 至少占本轮计划仓位预算的 30%。
- 若周线核心空单仍在，顺势日线空单属于同向加风险，必须重新计算总风险。

退出：

- 按日线空单自身止损止盈规则执行。
- 不影响周线核心空单的剩余仓位，除非触发账户级风控。

### 5.3 周线多趋势下的日线战术仓

多头环境对称：

- 周线多趋势下，日线死叉可作为日线回调空单。
- 日线回调失败并重新金叉后，可开顺势日线多单。
- 反向日线仓只能是战术仓，不能升级为周线核心仓。

## 6. 4H 执行层

4H 的作用是提高入场质量，而不是制造频繁交易。

考虑到 4H 趋势波段震荡频繁，且价格可能长时间处在很窄的 Bollinger Bands 区间内，若 4H 条件不清晰，宁可不开 4H 仓位。

### 6.1 4H 禁止交易状态

```text
IF 4H BOLL width contracting:
    H4 = NO_TRADE
```

禁止事项：

- 不开独立 4H 仓。
- 不用 4H 震荡信号频繁止盈止损。
- 不因为短线假突破打乱周线和日线仓位。

### 6.2 4H 允许执行状态

```text
IF 4H BOLL width expanding
AND 4H structure break confirmed:
    H4 = ENTRY_CONFIRM
```

允许事项：

- 为周线核心仓寻找更好的入场点。
- 为日线顺势仓寻找执行点。
- 在极严格条件下，允许小比例 4H 执行仓。

### 6.3 4H 独立仓位

若后续保留 4H 独立仓，应定义为：

```text
POSITION_TYPE = H4_EXECUTION_TRADE
```

仓位：

- 最多占本轮计划仓位预算的 10%。
- 只在 Bollinger Bands 开口、结构突破、风险收益比足够时开仓。

原则：

- BOLL 不开口，不开 4H 仓。
- 结构不确认，不开 4H 仓。
- 风险收益比不足，不开 4H 仓。
- 4H 仓不允许频繁补仓摊平。

## 7. 统一风险预算

基础预算：

```text
TOTAL_POSITION_BUDGET = 100%

WEEKLY_CORE = >= 60%
DAILY_TREND = >= 30%
H4_EXECUTION = <= 10%
```

该预算表示策略层级分配，不等于账户资金直接使用比例。

落地时必须同时计算：

- 单笔计划亏损。
- 总计划亏损。
- 保证金占用。
- 名义价值。
- 杠杆倍数。
- 强平价距离。
- 同向仓位总风险。
- 反向仓位净敞口。

### 7.1 同向仓位

周线空单 + 顺势日线空单属于同向风险。

开顺势日线空单前，必须确认：

- 周线核心仓仍符合持有条件。
- 日线顺势信号不是追空。
- 同向总计划风险没有超预算。
- 强平安全距离仍然足够。

### 7.2 反向仓位

周线空单 + 日线反弹多单属于反向战术仓。

反向仓不能被理解成“无风险对冲”。它可能降低净敞口，也可能增加手续费和波动损耗。

开反向仓前，必须确认：

- 它是日线战术仓，不是周线反转仓。
- 风险预算独立。
- 持仓周期更短。
- 退出条件更严格。

## 8. 状态切换优先级

交易系统必须有清晰优先级：

```text
账户级风控
> 强平风险
> 硬止损
> 周线核心趋势反转
> 周线 MA60 防线减仓
> 日线战术仓止损止盈
> 4H 禁止交易状态
> 新开仓
```

解释：

- 风控永远优先于策略。
- 已有仓位管理优先于新开仓。
- 周线核心仓管理优先于日线战术仓。
- 日线战术仓优先于 4H 机会仓。

## 9. 需要补充的细节定义

后续实现前，需要把以下词全部量化：

### 9.1 日线死叉

需要明确：

- 是否要求 Daily EMA15 收盘低于 Daily MA60。
- 是否要求连续 N 根日线确认。
- 是否要求 EMA15 斜率向下。
- 是否要求价格也在 MA60 下方。

### 9.2 日线反弹失败

可选定义：

- 日线金叉后重新死叉。
- 日线反弹未突破前高，并跌破最近 swing low。
- 日线收盘重新跌破 MA60。
- 反弹期间成交/波动无法延续，随后空头结构恢复。

### 9.3 周线站上 MA60

建议定义为：

- 周线收盘价站上 Weekly MA60。
- 或连续 N 根周线收盘在 Weekly MA60 上方。

不建议用周内价格刺破直接触发核心仓减半。

### 9.4 BOLL 开口

需要明确：

- 使用 4H Bollinger Bands width。
- width 是否高于过去 N 根均值。
- width 是否连续扩大。
- 是否要求价格突破上轨/下轨或突破结构。

### 9.5 结构突破确认

需要明确：

- swing high / swing low 的 lookback。
- 是收盘突破还是影线突破。
- 是否需要回踩确认。
- 突破后允许追单的最大距离。

---

## 10. 核心原则

- 周线是大环境，也是核心仓位层。
- 日线是战术层，只在周线环境中解释。
- 4H 是执行层，宁可错过，也不在窄幅震荡中频繁交易。
- 周线仓、日线仓、4H 仓必须有不同身份，不能混用。
- 顺势仓和反向仓必须分开管理。
- 仓位比例必须落到统一风险预算，不能变成简单加杠杆。
- 任何不确定状态，默认不开新仓。

---

## 11. 一句话定义

```text
A multi-timeframe position hierarchy:

Weekly = regime + core position
Daily  = tactical position inside weekly regime
H4     = strict execution filter
Risk   = unified budget and defense line
```
