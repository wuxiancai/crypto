# 策略触发条件分析

生成时间：2026-06-23  
关注问题：收益率低、胜率低是否与策略触发条件有关；截图红框中的“15m 反弹到 EMA50 区域”有什么作用；是否存在更优触发条件。

## 结论摘要

是的，当前胜率偏低很可能和触发条件有关，尤其是 15m 回踩/反弹条件偏宽。

截图红框里的条件并不是“已经反转成功”，只是说明 15m K 线碰到了快线附近的价格带。当前代码实际逻辑：

- 做多回踩：`low <= EMA快线 + ATR` 且 `close >= EMA快线 - ATR`
- 做空反弹：`high >= EMA快线 - ATR` 且 `close <= EMA快线 + ATR`

这等于把“快线附近上下 1 个 ATR 的区域”都算作触发区。它的作用是避免错过影线触达快线后的入场机会，但副作用是会把很多质量一般的回踩/反弹也放进来，导致止损次数多、胜率上不去。

另一个重要问题：页面文案硬编码写成“EMA50 区域”，但实际回测参数可能是 `EMA18 / MA40`、`EMA15 / MA60`、`EMA12 / MA50`。也就是说，截图里的“EMA50”更准确应显示为“当前快线区域”，例如 `EMA18 区域`。这是一个展示误导点。

## 红框条件的作用

红框里的“15m 反弹到 EMA50 区域”属于主趋势做空策略的一环。

做空策略的完整大意：

1. 4h 是空头结构。
2. 4h 空头动能确认。
3. 1h 是空头结构。
4. 1h 空头动能确认。
5. 15m 价格反弹到快线附近。
6. 15m 出现看跌确认。
7. 止损有效。
8. 风险收益比达标。

其中第 5 条“反弹到快线区域”的作用是：不在价格已经远离均线时追空，而是等价格回抽到均线附近再开空。这个思想本身是对的。

但当前实现的问题是：它只要求“碰到区域”，没有要求“反弹失败的质量足够好”。

## 当前条件为什么可能拉低胜率

### 1. 区域太宽

当前做空反弹条件：

```text
high >= EMA快线 - ATR
close <= EMA快线 + ATR
```

这个区域宽度接近 2 个 ATR。只要 15m K 线高点触到区域，收盘还在区域附近，就算满足。它不要求：

- 收盘必须重新跌回 EMA 快线下方。
- K 线上影线必须明显。
- 收盘必须接近 K 线低位。
- 当前 K 线实体必须足够强。
- 下一根 K 线继续确认。

所以很多“只是碰了一下均线区域、但趋势并未恢复”的 K 线也会触发。

### 2. 看跌/看涨确认太弱

当前看跌确认只是：

```text
close < open
```

当前看涨确认只是：

```text
close > open
```

这只是阴线/阳线，不等于高质量反转。小阴线、小阳线、十字附近实体都可能通过。

### 3. 风险收益比检查实际不够有区分度

当前主策略构造里，止盈是按 `target_risk_reward` 直接由止损距离算出来的，所以 RR 通常天然等于 2。也就是说，“风险收益比达标”更多是形式检查，不是真正基于上方/下方结构位判断是否有足够空间。

### 4. REVERSAL_PROBE 拖累明显

从 Ubuntu 数据库回测拆分看：

| Run | 组合 | TREND_PULLBACK | REVERSAL_PROBE |
| ---: | --- | --- | --- |
| 272 | EMA18 / MA40, 1y | 196 笔，胜率 44.90%，净收益 471.17 | 12 笔，胜率 16.67%，净收益 -38.79 |
| 140 | EMA15 / MA60, 1y | 164 笔，胜率 45.73%，净收益 435.34 | 5 笔，胜率 20.00%，净收益 -7.30 |
| 208 | EMA12 / MA50, 1y | 180 笔，胜率 46.67%，净收益 443.79 | 5 笔，胜率 0.00%，净收益 -21.50 |
| 150 | EMA15 / MA60, Swing30, 1y | 122 笔，胜率 46.72%，净收益 283.25 | 0 笔 |

这说明：当前主趋势策略本身还能赚钱，趋势转换试仓在这些优质组合里反而拖累胜率和净收益。

## 更优触发条件建议

### A. 修正展示文案

页面条件不应写死“EMA50 区域”，应根据当前配置显示：

- `EMA18 反弹到快线区域`
- `EMA15 回踩到快线区域`
- `MA/EMA 快线区域`

否则用户会以为实际策略一直在看 EMA50，但回测最佳组合实际用的是 EMA18、EMA15、EMA12 等快线。

### B. 收窄快线触发区域

把当前 `±1 ATR` 区域收窄，例如：

```text
做空：
high >= fast_ma - 0.5 * ATR
close <= fast_ma

做多：
low <= fast_ma + 0.5 * ATR
close >= fast_ma
```

目的：仍允许影线触达，但要求收盘重新回到趋势方向侧。

### C. 增加 K 线拒绝质量

做空建议：

```text
close < open
close <= low + 0.4 * (high - low)
upper_wick >= body * 0.5
```

做多建议：

```text
close > open
close >= high - 0.4 * (high - low)
lower_wick >= body * 0.5
```

目的：区分“只是小阴小阳”与“真正从快线区域被打回”。

### D. 增加趋势强度门槛

当前默认 `min_adx=20`。建议回测：

- `ADX >= 22`
- `ADX >= 25`
- `ADX >= 28`

同时加入 DI 差值过滤：

```text
做多：DI+ - DI- >= 5
做空：DI- - DI+ >= 5
```

目的：过滤震荡行情里的均线反复穿越。

### E. 提高 RR 或改为结构空间 RR

当前 RR 基本等于配置的目标 RR，不够有筛选力。建议改成结构空间判断：

```text
做多：上方最近结构阻力 / 入场风险 >= 2.2 或 2.5
做空：下方最近结构支撑 / 入场风险 >= 2.2 或 2.5
```

如果暂时不做结构阻力/支撑，至少先回测：

- `min_risk_reward=2.0`
- `min_risk_reward=2.2`
- `min_risk_reward=2.5`

### F. 暂停或严格限制 REVERSAL_PROBE

建议先回测：

1. `TREND_PULLBACK only`
2. `REVERSAL_PROBE only`
3. `REVERSAL_PROBE score >= 80`
4. `REVERSAL_PROBE score >= 85`
5. `REVERSAL_PROBE 只做 CONFIRMED，不做 EARLY`

当前数据看，先禁用 REVERSAL_PROBE 很可能提升胜率和净收益稳定性。

## 第一批应回测的新触发条件组合

基于 1 年口径 Top 组合，建议优先测试：

| 编号 | 基础参数 | 新触发条件 |
| ---: | --- | --- |
| 1 | EMA18 / MA40, ATR12, DMI12, Swing20 | TREND_PULLBACK only + close 必须回到快线方向侧 |
| 2 | EMA18 / MA40, ATR12, DMI12, Swing20 | 条件 1 + ADX>=25 + DI gap>=5 |
| 3 | EMA18 / MA40, ATR12, DMI12, Swing20 | 条件 2 + K 线拒绝质量过滤 |
| 4 | EMA15 / MA60, ATR14, DMI12, Swing20 | TREND_PULLBACK only + close 必须回到快线方向侧 |
| 5 | EMA15 / MA60, ATR12, DMI12, Swing30 | 条件 4 + ADX>=25 + DI gap>=5 |
| 6 | EMA12 / MA50, ATR12, DMI12, Swing20 | TREND_PULLBACK only + K 线拒绝质量过滤 |

预期：

- 交易次数会下降。
- 胜率应上升。
- 盈亏比可能略下降或持平。
- 如果过滤太严，收益率会下降。

目标不应是单纯“胜率最高”，而是：

- 胜率 > 50% 作为第一阶段目标。
- 利润因子 >= 1.6。
- 盈亏比 >= 1.5。
- 最大回撤不高于当前 Run 140 / Run 150。

60% 胜率可以作为第二阶段目标，但大概率需要明显减少交易数量，甚至需要更高周期确认或分侧策略。

## 最终判断

当前红框条件是有作用的：它防止追空/追多，要求价格回到快线附近再交易。

但当前实现确实偏宽，尤其“触达快线 ± ATR 区域 + 简单阴阳线确认”不足以保证高胜率。更优方向不是删除这个条件，而是把它升级为“高质量回踩/反弹失败确认”：

1. 快线区域收窄。
2. 收盘必须回到趋势方向侧。
3. 增加影线/实体质量。
4. 提高 ADX/DI 动能门槛。
5. 暂停或提高 REVERSAL_PROBE 门槛。

这比继续盲目调 EMA 周期更可能提升胜率和盈亏比。
