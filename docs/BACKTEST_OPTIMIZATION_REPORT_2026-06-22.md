- 最大回撤：38.95，约 3.31%

这不是高胜率组合，而是典型的“中等偏低胜率 + 较高盈亏比”组合。它能赚钱的核心不是胜率，而是盈利单平均收益约为亏损单平均亏损的 2.08 倍。

## 胜率与盈亏比观察

胜率最高的正收益组合并没有超过 47%。当前策略更适合按“盈亏比 + 利润因子 + 回撤”评估，而不是追求 60%-80% 胜率。

| 维度 | 最优组合 | 结论 |
| --- | --- | --- |
| 最高胜率 | Run 150，46.72% | `Swing30` 降低交易次数，胜率和回撤更好，但收益低于 Swing20 |
| 最高盈亏比 | Run 148，2.11 | `ATR12 + DMI12 + Swing20` 的盈利单更肥，但最终收益略低于 Run 140 |
| 最高利润因子 | Run 150，1.71 | 风险收益最均衡，适合作为更保守候选 |
| 最高收益 | Run 140，42.80% | 适合作为进攻候选，但需要防止过拟合 |

## 不建议继续押注的区域

慢参数整体较弱，尤其快线 45-50 与慢线 120-180 的组合：

- `EMA50 / MA150`：净收益 -7.83，胜率 35.75%，最大回撤约 10.88%。
- `EMA50 / MA180`：净收益 4.32，胜率 35.18%，最大回撤约 10.50%。
- `EMA50 / MA90`：净收益 8.68，胜率 35.34%，交易 249 笔，手续费 177.66。

这说明慢快线组合会更频繁地产生低质量交易，手续费吞噬明显，且回撤扩大。后续优化应优先围绕快线 10-20、慢线 50-100，而不是继续扩大到 150/180/200。

### 成本过滤验证

当前 `fee/risk=0` 在收益上占优，但真实交易中关闭成本过滤可能放大滑点和极端费率风险。建议追加：

- `max_fee_to_risk_ratio=0.05`
- `max_fee_to_risk_ratio=0.10`
- `max_fee_to_risk_ratio=0.15`
- `max_fee_to_risk_ratio=0.20`
- `max_fee_to_risk_ratio=0.25`

重点比较 `EMA15 / MA60` 与 `EMA15 / MA90`，看是否存在收益接近 Run 140、但成本暴露低于 Run 140 的中间值。

## 验收建议

在进入长期 Paper Trading 前，建议把候选组合按三档并行跟踪：

1. 进攻档：Run 140 参数，追求最高收益。
2. 平衡档：Run 150 参数，追求较高胜率、利润因子和低回撤。
3. 防守档：Run 134 参数，保留手续费/风险过滤。

每档至少追加：

- BTCUSDT 最近 2 年回测。
- ETHUSDT 最近 1 年与 2 年回测。
- 最近 3 个月、6 个月滚动窗口回测。
- 连续 Paper Trading 观察，记录真实滑点、拒单、资金费率和极端行情表现。

当前结论只证明这些组合在 BTCUSDT 最近 1 年历史数据中表现较好，还不能直接作为实盘参数冻结依据。
